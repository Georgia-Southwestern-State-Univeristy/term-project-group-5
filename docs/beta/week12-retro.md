## Wellness Check

The experiences that went well in reaching Beta are:
1. Incorporating testing throughout our feature implementations instead of at the end of a feature's completion. 
2. Delivery discipline by ensuring all weekly deliverables are completed by their deadline and each team member completes the requirements that they own.
3. Staying true to using the architecture that we agreed to in our project proposal.
4. Everyone on the team stepping up to help and provide support when another team member is stuck or lacks resources for the development that they own.

There are two things that slowed the team down:
1. A breakdown in scope control that allowed scope creep to occur. This scope creep appeared as shifting goals and expectations on the features that were agreed to and documented in our project proposal. This resulted in misalignment, delay, and rework of our primary flight finder feature.
2. Not consistently following documented processes regarding ownership of tasks and PR.


## Lessons Learned
Top 3 lessons learned:
1. A wireframe of the UI for each feature should be included in the project propsal along with a detailed explanation of the scope for each feature.
2. Documented expectations and processes should be included in the team charter at the start of the project regarding how frontend and backend developers are to work together in regards to implementations and changes to code fixes.
3. Unit testing and integration testing should be continuously implemented throughout backend development, especially after each code change is made.
