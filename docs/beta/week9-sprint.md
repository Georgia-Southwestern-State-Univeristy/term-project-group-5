Sprint Goal:

By Friday, users can search flights based off their preferences reliably.

Committed items(Ranked):

1. Integrate Amadeus API for submitting flight searches

- Acceptance criteria: Application can make successful get requests to the Amadeus FlightSearchOffer API and receive results
- Owner: Keenan

2. Create page/popup for flight results

- Acceptance criteria: Website can display flight offers after submitting a request. Should be seperate from destination list
- Owner: Quynh

3. User registration

- Acceptance criteria: Users can register using personal information. Should not be able to create more than one account using the same information.
- Owner: Keenan

4. User authentication

- Acceptance criteria: Application can verify users have created an account and are who they say they are.
- Owner: Keenan

5. Login/Registration page

- Acceptance criteria: Website displays a small form on a page so users can fill out information to register or login.
- Owner: Zengtao

6. Previous search retrieval by user

- Acceptance criteria: Users can recall previous searches to not have to refill out information.
- Owner: Keenan/Zengtao

7. Implement middleware

- Acceptance criteria: A middleware function intercepts requests to protected routes.
- Owner: Keenan

8. Rate limiting

- Acceptance criteria: Users are limited to how many requests they can make through our site within a certain amount of time.
- Owner: Keenan

9. UI optimization

- Acceptance criteria: Website subjectively performs or looks better.
- Owner: Quynh

10. Backend optimization

- Acceptance criteria: Website performs better or has stronger features in the backend.
- Owner: Keenan
