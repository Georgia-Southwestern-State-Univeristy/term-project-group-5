Date: March 17, 2026

1. Context
   - The application previously used the Amadeus Self-Service API for flight data. Amadeus is decommissioning its Developer Portal, rendering our current integration and API keys obsolete. We require a stable, real-time replacement for our MERN stack.

2. Key forces
   - Availability: Immediate access to a working developer portal.

   - Cost: Zero to low cost for testing and deployment

   - Compatibility: Support for IATA codes and round-trip queries via Node.js.

3. Considered Options
   - Flight Scraper Sky (RapidAPI): High availability, easy Axios integration, reliable data.

   - Skyscanner API: Industry standard, but has a high barrier for student API approval.

4. Decision
   - Chosen Option: Flight Scraper Sky (via RapidAPI)

5. Status: Accepted

6. Consequences:
   - Positive: Restores real-time functionality with minimal frontend refactoring.

   - Negative: Requires handling specific entityId formats (e.g., PARI for Paris) instead of standard 3-letter IATA codes in some cases.

   - Neutral: Shift from Amadeus SDK to standard axios requests.

7. Implementation Plan (Implemented as of March 15, 2026)
   - Backend: Update flightController.js to use the /flights/search-roundtrip endpoint.

   - Security: Replace AMADEUS_KEY with RAPIDAPI_KEY in .env and Docker configs.

   - Data Mapping: Maintain the existing FlightOffer JSON structure to avoid breaking the React frontend.
